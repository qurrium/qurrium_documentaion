For the write() function actually exports 4 different files respecting to adventure, legacy, tales, and reports like:

    'folder': './bla_exp/',

    'reports': './bla_exp/reports/bla_exp.id={exp_id}.reports.json',

write(multiprocess: bool = False, pbar: tqdm | None = None) → tuple[str, dict[str, str]][source]#

Export the experiment data, if there is a previous export, then will overwrite.

Hint

This function will traversal all objects in the export_set, so it will ensure the jsonable of all objects. And this will reduce the performance of exporting.

Parameters:

  • multiprocess (bool, optional) ‚Äì Whether to use multiprocess to export, Defaults to False. It‚Äôs dangerous to use multiprocess to export. It may cause memory leak.

  • pbar (Optional[tqdm.tqdm], optional) ‚Äì The progress bar for exporting. Defaults to None.

Returns:

The first element is the id of experiment, the second element is the dictionary of files of experiment.

Return type:

tuple[str, dict[str, str]]
